# v1.6.0 — Report-Verzeichnis-Isolation

Du bist der Team Lead eines Agent-Teams namens "codearch-v1.6". Dein Ziel: Das Report-Overwrite-Problem lösen, indem Report-Typ-spezifische Unterverzeichnisse eingeführt werden, und gleichzeitig die Möglichkeit schaffen, mehrere Report-Typen in einem Run zu generieren.

## Ausgangslage

PhpCodeArcheology generiert Reports in verschiedenen Formaten (HTML, Markdown, JSON, SARIF, AI-Summary). Aktuell schreiben alle Reports ins selbe Verzeichnis (`reportDir`). HTML und Markdown rufen `clearReportDir()` auf und löschen dabei **alle** vorherigen Reports. JSON/SARIF/AI-Summary vermischen sich mit HTML-Assets.

### Relevante Architektur

```
Visitors (src/Analysis/) → sammeln Rohdaten
  ↓
Calculators (src/Calculators/) → berechnen Metriken
  ↓
MetricsController → zentraler Daten-Hub
  ↓
DataProviders (src/Report/DataProvider/) → formatieren für Templates
  ↓
Reports (src/Report/) → rendern Output
```

### Betroffene Dateien

| Datei | Rolle | Änderung |
|-------|-------|----------|
| `src/Report/ReportTrait.php` | Shared Trait mit `clearReportDir()`, `deleteDirContents()`, `renderTemplate()` | `clearReportDir()` auf Unterverzeichnis umstellen |
| `src/Report/HtmlReport.php` | HTML-Report mit vielen Seiten, nutzt Twig-Templates | Unterverzeichnis `html/` nutzen |
| `src/Report/MarkdownReport.php` | Markdown-Report | Unterverzeichnis `markdown/` nutzen |
| `src/Report/JsonReport.php` | JSON-Report mit `buildProjectSection()`, `buildFilesSection()` etc. | Unterverzeichnis `json/` nutzen |
| `src/Report/SarifReport.php` | SARIF-Format-Report | Unterverzeichnis `sarif/` nutzen |
| `src/Report/AiSummaryReport.php` | AI-Summary-Report | Unterverzeichnis `ai-summary/` nutzen |
| `src/Report/ReportFactory.php` | Factory, erstellt Reports via `create()` | Mehrfach-Report-Logik (Array statt einzelner Typ) |
| `src/Report/ReportInterface.php` | Interface mit `generate()` Methode | Prüfen ob Signatur angepasst werden muss |
| `src/Application/Application.php` | Entry Point, `run()`, `runAnalysis()`, orchestriert Report-Generierung | Mehrere Reports nacheinander generieren |
| `src/Application/ArgumentParser.php` | Parst CLI-Args inkl. `--report-type` | Komma-separierte Report-Types parsen |
| `src/Application/Config.php` | Config-Verwaltung, validiert Report-Typen | Multi-Type-Validierung |

### Unterverzeichnis-Struktur (Ziel)

```
reportDir/
├── html/              ← HTML-Report (index.html, single-*.html, assets/)
├── markdown/          ← Markdown-Report
├── json/              ← report.json
├── sarif/             ← report.sarif.json
├── ai-summary/        ← ai-summary.md
├── graph/             ← (reserviert für v1.7.0)
└── history.jsonl      ← Shared über alle Report-Typen
```

### Teststruktur

- Tests: Pest PHP (`tests/Unit/`, `tests/Feature/`)
- Ausführen: `./vendor/bin/pest`
- Bestehende Report-Tests: `tests/Unit/Report/`
