# v1.7.0 — Knowledge Graph JSON-Export

Du bist der Team Lead eines Agent-Teams namens "codearch-v1.7". Dein Ziel: Einen neuen Report-Typ `--report-type=graph` implementieren, der die Codebase-Struktur als Knowledge Graph (Nodes + Edges) in JSON exportiert. Maschinenlesbar für AI-Tools.

## Voraussetzung

**v1.6.0 muss abgeschlossen sein.** Der Graph-Report nutzt das Unterverzeichnis `reportDir/graph/`.

## Ausgangslage

### Architektur-Pipeline

```
Visitors (src/Analysis/) → sammeln Rohdaten
  ↓
Calculators (src/Calculators/) → berechnen abgeleitete Metriken
  ↓
MetricsController → zentraler Daten-Hub (Collections für Files, Classes, Functions, Packages)
  ↓
DataProviders (src/Report/DataProvider/) → formatieren für Templates/Reports
  ↓
Reports (src/Report/) → rendern Output
```

### Vorhandene Datenquellen für den Graph

| Quelle | Datei | Liefert |
|--------|-------|---------|
| DependencyVisitor | `src/Analysis/DependencyVisitor.php` | `classDependencies`, `classUses`, `classTraits`, `functionDependencies`, `methodDependencies`, `outsideDependencies` |
| IdentifyVisitor | `src/Analysis/IdentifyVisitor.php` | Klassen-Identifikation (extends, implements, traits) |
| DependencyCycleCalculator | `src/Calculators/DependencyCycleCalculator.php` | Zyklen via Tarjan-SCC-Algorithmus |
| CouplingCalculator | `src/Calculators/CouplingCalculator.php` | Afferent/Efferent Coupling |
| GitAnalyzer | `src/Git/GitAnalyzer.php` | Commits, Autoren, File-Changes |
| PackageVisitor | `src/Analysis/PackageVisitor.php` | Package/Namespace-Zuordnung |
| MetricsController | `src/Metrics/MetricsController.php` | Alle Collections (file, class, function, package) |

### Bestehende DataProviders (als Referenz)

| Provider | Liefert |
|----------|---------|
| `ClassDataProvider` | Klassen mit allen Metriken |
| `ClassCouplingDataProvider` | Coupling-Beziehungen |
| `PackagesDataProvider` | Package-Struktur |
| `GitDataProvider` | Git-Metriken pro Datei |
| `RefactoringPriorityDataProvider` | Priorisierte Refactoring-Kandidaten |
| `DataProviderFactory` | Zentrale Factory für alle Provider |

### Ziel-JSON-Struktur

```json
{
  "version": "1.0",
  "generatedAt": "2026-03-24T12:00:00+00:00",
  "nodes": [
    { "id": "class:App\\UserService", "type": "class", "name": "UserService",
      "path": "src/UserService.php", "metrics": { "cc": 12, "lcom": 3, "mi": 65 },
      "problems": [{ "level": "warning", "message": "High cyclomatic complexity" }] }
  ],
  "edges": [
    { "source": "class:App\\UserService", "target": "class:App\\UserRepository",
      "type": "depends_on", "weight": 1 }
  ],
  "clusters": [
    { "id": "package:App\\Services", "name": "App\\Services", "nodeIds": ["class:App\\UserService"] }
  ],
  "cycles": [
    { "nodes": ["class:A", "class:B", "class:C"], "length": 3 }
  ]
}
```

### Node-Typen

| Typ | ID-Prefix | Metriken |
|-----|-----------|----------|
| `file` | `file:` | loc, lloc, cc, mi |
| `class` | `class:` | cc, lcom, mi, instability, afferentCoupling, efferentCoupling |
| `function` | `function:` | cc, cogC, params |
| `package` | `package:` | cohesion, abstractness, instability |
| `author` | `author:` | commitCount, filesChanged |

### Edge-Typen

| Typ | Bedeutung | Quelle |
|-----|-----------|--------|
| `contains` | File→Class, Class→Method, Package→Class | IdentifyVisitor |
| `extends` | Class→Parent | IdentifyVisitor |
| `implements` | Class→Interface | IdentifyVisitor |
| `uses_trait` | Class→Trait | IdentifyVisitor |
| `depends_on` | Class→Class (Dependency) | DependencyVisitor |
| `belongs_to` | Class/File→Package | PackageVisitor |
| `authored_by` | File→Author | GitAnalyzer |
| `cycle_member` | Class↔Class (bidirektional) | DependencyCycleCalculator |
