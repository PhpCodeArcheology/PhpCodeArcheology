# v1.8.0 — D3 Knowledge Graph Visualisierung

Du bist der Team Lead eines Agent-Teams namens "codearch-v1.8". Dein Ziel: Eine interaktive D3.js Force-Directed Graph Visualisierung der Codebase in den HTML-Report integrieren. Eine neue Seite "Knowledge Graph" mit Filter-Controls, Zoom, Tooltips und Cycle-Highlighting.

## Voraussetzung

**v1.7.0 muss abgeschlossen sein.** Die Graph-Daten (`GraphDataProvider`, `graph.json`) sind bereits verfügbar.

## Ausgangslage

### HTML-Report-System

Der HTML-Report wird von `HtmlReport.php` generiert und nutzt Twig-Templates:

```
src/Report/HtmlReport.php          → Orchestriert alle Seiten
templates/html/                     → Twig-Templates
templates/html/parts/               → Wiederverwendbare Parts (header, navi, footer)
templates/html/assets/              → CSS, JS, Fonts
```

### Bestehende Seiten im HTML-Report

| Seite | Template | Methode in HtmlReport.php |
|-------|----------|--------------------------|
| Dashboard | `index.html.twig` | `generateIndexPage()` |
| Files | `files.html.twig` | `generateFilePage()` |
| Classes | `classes.html.twig` | `generateClassPage()` |
| Class Coupling | `class-coupling.html.twig` | `generateClassCouplingPage()` |
| Classes Chart | `classes-chart.html.twig` | `generateClassChartPage()` |
| Functions | `functions-list.html.twig` | `generateFunctionPage()` |
| Packages | `packages-list.html.twig` | `generatePackagesPage()` |
| Problems | `problems.html.twig` | `generateProblemsPage()` |
| Refactoring Roadmap | `refactoring-roadmap.html.twig` | `generateRefactoringRoadmapPage()` |
| Git | `git.html.twig` | `generateGitPage()` |
| Glossary | `glossary.html.twig` | `generateGlossaryPage()` |

### Bestehende JS-Libraries im Report

- `templates/html/assets/js/chart.umd.min.js` — Chart.js
- `templates/html/assets/js/mermaid.min.js` — Mermaid Diagramme
- `templates/html/assets/js/svg-pan-zoom.min.js` — SVG Pan/Zoom
- `templates/html/assets/js/sortable.min.js` — Tabellen-Sortierung
- `templates/html/assets/js/main.js` — Custom JS

### Design-System

- CSS: Tailwind-basiert (`assets/css/output.css`)
- Fonts: Oswald + Source Sans 3 (`assets/fonts/`)
- Dark/Light Theme Support (bestehend)
- Navigation: `parts/navi.html.twig`

### Graph-Daten (aus v1.7.0)

Der `GraphDataProvider` liefert:
- `nodes[]` — mit id, type, name, path, metrics, problems
- `edges[]` — mit source, target, type, weight
- `clusters[]` — Package-Gruppierungen
- `cycles[]` — Dependency-Zyklen
